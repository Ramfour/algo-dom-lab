## Этап 5. Selection sort — сортировка выбором

Цель: научиться искать минимум в неотсортированной части массива.

### Идея

Массив делится на два набора: левый (отсортированный) и правый (неотсортированный). На каждом шаге мы находим минимальный элемент в правой части и ставим его в конец левой. Никаких сдвигов — только один обмен за проход.

### Что нужно понимать

- Почему внешний цикл идёт до `arr.length - 1`.
- Почему внутренний цикл начинается с `i + 1`.
- Зачем сохранять `minIndex`, а не сразу менять местами.
- Сложность всегда `O(n²)` — флаг оптимизации не поможет (всё равно ищем минимум каждый раз).
- Отличие от Bubble Sort: делаем один обмен за проход, а не много.

### Главная схема

```js
for (let i = 0; i < arr.length - 1; i++) {
  let minIndex = i;
  for (let j = i + 1; j < arr.length; j++) {
    if (arr[j] < arr[minIndex]) {
      minIndex = j;
    }
  }
  if (minIndex !== i) {
    [arr[i], arr[minIndex]] = [arr[minIndex], arr[i]];
  }
}
```

### Задания

1. Напиши `selectionSort(arr)` — принимает массив чисел, сортирует по возрастанию, возвращает отсортированный массив.
2. Напиши `selectionSort(arr, compare)` — то же самое, но правило сравнения передаётся как функция `compare(a, b)`.
3. Вынеси поиск индекса минимума в отдельную функцию `findMinIndexFrom(arr, startIndex, compare)` — принимает массив, стартовый индекс и компаратор, возвращает индекс минимального элемента в диапазоне от `startIndex` до конца массива.
4. Прогони руками `[6, 3, 1, 7]` — запиши, какой элемент выбирается как минимум на каждом шаге и куда он ставится.
5. Отсортируй `fruits` по `weight` по возрастанию, передав в сортировку компаратор `compareByWeightAsc`.

## Этап 6. Insertion sort — сортировка вставками

Цель: понять сдвиги элементов и вставку в правильное место.

### Идея

Левая часть массива считается отсортированной. Берём следующий элемент, сохраняем его в `current`, сдвигаем вправо все элементы, которые больше него, и вставляем `current` на освободившееся место.

### Главная схема

```js
for (let i = 1; i < arr.length; i++) {
  const current = arr[i];
  let j = i - 1;

  while (j >= 0 && arr[j] > current) {
    arr[j + 1] = arr[j];
    j--;
  }

  arr[j + 1] = current;
}
```

### Задания

## Этап 9. Quick sort — быстрая сортировка

Цель: понять `pivot`, разделение массива и рекурсию.

### Идея

Выбираем опорный элемент `pivot` (обычно последний или средний). Все элементы меньше него кладём в `left`, все больше — в `right`. Потом рекурсивно сортируем `left` и `right` и собираем: `[...quickSort(left), pivot, ...quickSort(right)]`.

### Что нужно понимать

- Что такое `pivot` и как его выбирать.
- Почему базовый случай: массив из 0 или 1 элемента.
- Средняя сложность `O(n log n)`, худший случай `O(n²)` (если pivot всегда минимальный или максимальный).
- Отличие от Merge Sort: не требует доп. памяти для слияния, но нестабильная.
- Рекурсия: функция вызывает саму себя с уменьшенным массивом.

### Главная схема

```js
function quickSort(arr) {
  if (arr.length <= 1) return arr;  // базовый случай

  const pivot = arr[arr.length - 1];
  const left = [];
  const right = [];

  for (let i = 0; i < arr.length - 1; i++) {
    if (arr[i] <= pivot) left.push(arr[i]);
    else right.push(arr[i]);
  }

  return [...quickSort(left), pivot, ...quickSort(right)];
}
```

### Задания

1. Напиши `quickSort(arr)` — принимает массив чисел, возвращает новый отсортированный по возрастанию массив. Используй подход с созданием `left` и `right` массивов.
2. Напиши `quickSort(arr, compare)` — то же самое, но правило сравнения передаётся как функция `compare(a, b)`.
3. Объясни базовый случай — при каком условии функция возвращает результат без рекурсии?
4. Объясни, что такое `pivot` — как его выбор влияет на эффективность сортировки?

## Этап 10. Merge sort — сортировка слиянием

Цель: понять алгоритм "разделяй и властвуй".

### Идея

Две фазы: **разделение** и **слияние**. Делим массив пополам рекурсивно, пока не получим массивы из одного элемента (они уже отсортированы). Потом сливаем два отсортированных массива в один: берём меньший элемент из начала каждого и добавляем в результат.

### Что нужно понимать

- Базовый случай: массив из 0 или 1 элемента.
- Функция `merge(left, right)` — отдельная и самая важная часть.
- Сложность всегда `O(n log n)` — стабильнее Quick Sort.
- Требует доп. памяти `O(n)` для хранения левой и правой части.
- Стабильная сортировка: равные элементы сохраняют исходный порядок.

### Главная схема

```js
function merge(left, right) {
  const result = [];
  let l = 0, r = 0;
  while (l < left.length && r < right.length) {
    if (left[l] <= right[r]) result.push(left[l++]);
    else result.push(right[r++]);
  }
  return [...result, ...left.slice(l), ...right.slice(r)];
}

function mergeSort(arr) {
  if (arr.length <= 1) return arr;  // базовый случай
  const mid = Math.floor(arr.length / 2);
  const left = mergeSort(arr.slice(0, mid));
  const right = mergeSort(arr.slice(mid));
  return merge(left, right);
}
```

### Задания

1. Напиши `merge(left, right)` — принимает два уже отсортированных массива, возвращает один новый массив, в котором все элементы обоих массивов стоят в правильном порядке.
2. Напиши `mergeSort(arr)` — принимает массив чисел, делит его пополам, рекурсивно сортирует каждую часть, сливает через `merge`, возвращает отсортированный массив.
3. Напиши `mergeSort(arr, compare)` — то же самое, но правило сравнения передаётся как функция `compare(a, b)`.
4. Объясни отличие `mergeSort` от `quickSort` — по памяти, стабильности и гарантированной сложности.
5. Прогони руками `[6, 3, 1, 7]` — нарисуй дерево разбиения и покажи, как происходит слияние.

Есть окно `[left, right]`, которое двигается по массиву. Мы расширяем или сужаем его, сохраняя текущее состояние.

### Задания

1. Напиши `maxSumSubarray(arr, k)` — принимает массив чисел и число `k`, возвращает максимальную сумму среди всех подмассивов длиной ровно `k`. Например, `maxSumSubarray([1,4,2,9,7,3], 3)` → `18`.
2. Напиши `averageSubarray(arr, k)` — принимает массив чисел и число `k`, возвращает массив средних значений для каждого окна длиной `k`. Например, `averageSubarray([1,3,2,6], 2)` → `[2, 2.5, 4]`.
3. Напиши `longestUniqueSubstring(str)` — принимает строку, возвращает длину самой длинной подстроки, в которой нет повторяющихся символов. Например, `'abcabcbb'` → `3` (`'abc'`).
4. Объясни, почему это лучше вложенного цикла — как окно позволяет избежать повторных вычислений?

## Этап 14. Stack — стек

2. Напиши `isValidBrackets(str)` — принимает строку со скобками `()[]{}`, возвращает `true`, если все скобки правильно открыты и закрыты в правильном порядке. Например, `'([{}])'` → `true`, `'([)]'` → `false`.
3. Напиши `reverseWithStack(str)` — принимает строку, возвращает её в обратном порядке, используя стек (не `split/reverse/join`).

## Этап 19. Big O

Цель: научиться оценивать скорость алгоритма.

### Обязательные сложности

- `O(1)` — константное время.
- `O(log n)` — логарифмическое время.
- `O(n)` — линейное время.
- `O(n log n)` — эффективные сортировки.
- `O(n²)` — вложенные циклы.

### Задания

1. Определи сложность `findMin` — сколько операций выполняется для массива из `n` элементов?
2. Определи сложность `bubbleSort` — почему два вложенных цикла дают `O(n²)`?
3. Определи сложность `binarySearch` — почему каждый шаг отбрасывает половину, и что это даёт?
4. Определи сложность `quickSort` в среднем случае и объясни, почему в худшем случае это `O(n²)`.
5. Определи сложность двойного цикла — посмотри на свои функции и найди все, где один цикл внутри другого.
6. Для каждой своей написанной функции добавь комментарий с её сложностью, например: `// O(n)`.

## Главное правило

Если алгоритм не получается, не пытайся сразу писать код. Сначала сделай трассировку руками:

```txt
массив: [6, 3, 1]
i = ?
j = ?
current = ?
что сравниваем?
что сдвигаем или меняем?
как выглядит массив после шага?
```

Когда ты можешь объяснить каждый шаг словами, код становится намного проще.
