# Итераторы и генераторы в JavaScript

## 1. Проблема, которую решают итераторы

У тебя есть массив из миллиона элементов:

```js
const bigArray = new Array(1000000).fill(0).map((_, i) => i);
```

**Что происходит при `for (const item of bigArray)`?**

Весь массив уже лежит в памяти. Цикл просто идёт по нему. Это нормально.

А теперь представь: **ты читаешь файл построчно**. Файл на 10 гигабайт — в память не влезет.

Как быть? Читать по одной строке, обработать, выкинуть, читать следующую.

Вот тут и нужен **итератор** — штука, которая выдаёт значения **по одному, по требованию**, а не все сразу.

---

## 2. Практический пример: чтение файла

```js
// Псевдокод чтения огромного файла
function* readFileLines(filename) {
  // открываем файл
  while (не конец файла) {
    yield прочитатьСледующуюСтроку();  // ← выдаём одну строку и ждём
  }
}

const lines = readFileLines('huge.txt');

for (const line of lines) {
  console.log(line);  // в памяти всегда только одна строка!
}
```

**Главное:** файл не загружается целиком. Ты читаешь кусочками.

---

## 3. Сравнение: массив vs итератор

| | Массив | Итератор |
|---|---|---|
| **Всё в памяти?** | Да, сразу | Нет, по частям |
| **Можно пройти повторно?** | Да | Нет (обычно), один проход |
| **Длина известна?** | Да (`length`) | Нет заранее |
| **Можно по индексу?** | Да (`arr[5]`) | Нет |

## 4. Как это выглядит в коде

### Массив (всё уже есть):
```js
const arr = [1, 2, 3, 4, 5];

for (const x of arr) {
  console.log(x);  // 1, 2, 3, 4, 5
}

// Можешь ещё раз:
for (const x of arr) {
  console.log(x);  // 1, 2, 3, 4, 5 снова работает
}
```

### Итератор (генератор — один из видов итератора):
```js
function* countToFive() {
  yield 1;
  yield 2;
  yield 3;
  yield 4;
  yield 5;
}

const iterator = countToFive();

for (const x of iterator) {
  console.log(x);  // 1, 2, 3, 4, 5
}

// Попробуем ещё раз:
for (const x of iterator) {
  console.log(x);  // ничего! итератор уже исчерпан
}
```

## 5. Аналогия из жизни

**Массив** — это список покупок на бумаге. Весь перед тобой, можешь смотреть на любой пункт.

**Итератор** — это человек, который стоит за дверью и передаёт тебе коробки **по одной**:
- Ты кричишь: "Следующая!"
- Он передаёт коробку
- Ты обработал, кричишь снова
- Когда коробки кончились — он молчит

---

## 6. Что такое генератор

Генератор — это функция, которая **приостанавливает** своё выполнение, отдаёт значение, и **может продолжить** с того же места при следующем вызове.

**PHP:**
```php
function countToFive() {
    yield 1;
    yield 2;
    yield 3;
    yield 4;
    yield 5;
}

foreach (countToFive() as $num) {
    echo $num;  // 1, 2, 3, 4, 5
}
```

**JS — то же самое:**
```js
function* countToFive() {
  yield 1;
  yield 2;
  yield 3;
  yield 4;
  yield 5;
}

for (const num of countToFive()) {
  console.log(num);  // 1, 2, 3, 4, 5
}
```

---

## 7. Что такое yield

`yield` — это **точка остановки** функции.

Когда генератор встречает `yield`, он:
1. Возвращает значение
2. **Замораживает** своё состояние (все переменные, позиция в коде)
3. Ждёт, когда его попросят продолжить

Пример с состоянием:

```js
function* counter() {
  let i = 0;
  
  while (true) {
    yield i++;  // ← остановился здесь, запомнил что i уже увеличено
  }
}

const gen = counter();

console.log(gen.next());  // { value: 0, done: false }
console.log(gen.next());  // { value: 1, done: false }
console.log(gen.next());  // { value: 2, done: false }
// Функция не запускается заново — она продолжает с места остановки!
```

## 8. Как это работает под капотом (state machine)

Когда ты пишешь:
```js
function* gen() {
  yield "A";
  yield "B";
  yield "C";
}
```

JS превращает это в **state machine** примерно такого вида:

```js
{
  state: 0,           // на какой строчке остановились
  localVars: {},      // локальные переменные
  
  next() {
    switch (this.state) {
      case 0:
        this.state = 1;  // запомнили: следующий раз начнём с case 1
        return { value: "A", done: false };
        
      case 1:
        this.state = 2;  // запомнили: следующий раз начнём с case 2
        return { value: "B", done: false };
        
      case 2:
        this.state = 3;
        return { value: "C", done: false };
        
      default:
        return { value: undefined, done: true };
    }
  }
}
```

**Нет узлов. Нет `next`. Есть только число `state` и `switch`.**

## 9. Итератор vs Связный список

### Связный список (Linked List)
```
[data: A, next: *] → [data: B, next: *] → [data: C, next: null]
     ↑                   ↑                   ↑
   узел 1             узел 2              узел 3
```

- Каждый узел **хранит ссылку** на следующий
- Узлы разбросаны в памяти
- Чтобы пройтись, нужно следовать по ссылкам `node.next`

### Массив + Итератор
```
┌─────────────────────────────┐
│  0     1     2     3      │  ← данные лежат в одном блоке памяти
│ [A]   [B]   [C]   [D]    │
└─────────────────────────────┘
           ↑
    итератор: position = 2
```

- Итератор хранит только **число** (индекс)
- Вычисляет `array[position]` при вызове `next()`
- Никаких ссылок между элементами нет

| | Связный список | Итератор в JS |
|---|---|---|
| Узлы связаны ссылками | Да | Нет |
| Данные хранятся в узлах | Да | Вычисляются "на лету" |
| Проход | `current = current.next` | `iterator.next()` — функция меняет внутреннее состояние |
| Память | пропорционально количеству элементов | константная (храним только state) |

## 10. Метод .next()

Когда ты вызываешь `function*`, она не выполняется. Она возвращает **объект-итератор**:

```js
function* example() {
  yield "A";
  yield "B";
  yield "C";
}

const iterator = example();  // функция ещё не выполнилась!

// У iterator есть метод .next()
console.log(iterator.next());  // { value: "A", done: false }
console.log(iterator.next());  // { value: "B", done: false }
console.log(iterator.next());  // { value: "C", done: false }
console.log(iterator.next());  // { value: undefined, done: true }
```

**Структура объекта, который возвращает `.next()`:**

| Свойство | Значение |
|----------|----------|
| `value` | Значение после `yield` |
| `done` | `false` — ещё есть что возвращать, `true` — генератор закончился |

## 11. Практические примеры итераторов в JS

```js
// 1. Генераторы (function*)
function* myGen() { yield 1; yield 2; }

// 2. Map.keys(), Map.values(), Map.entries()
const map = new Map([["a", 1], ["b", 2]]);
for (const key of map.keys()) { ... }

// 3. Set
const set = new Set([1, 2, 3]);
for (const x of set) { ... }

// 4. Строка (по символам)
for (const char of "hello") { ... }

// 5. NodeList из DOM
const divs = document.querySelectorAll("div");
for (const div of divs) { ... }  // это итерируемое, не массив!
```

## 12. Чтение большого файла (сравнение с PHP)

**PHP:**
```php
function readHugeFile($filename) {
    $handle = fopen($filename, "r");
    while (!feof($handle)) {
        yield fgets($handle);  // одна строка за раз
    }
    fclose($handle);
}

// Файл на 10 ГБ, но памяти хватит
foreach (readHugeFile("big.txt") as $line) {
    process($line);
}
```

**JS (аналогично):**
```js
async function* readHugeFile(filename) {
    const response = await fetch(filename);
    const reader = response.body.getReader();
    
    while (true) {
        const { done, value } = await reader.read();
        if (done) break;
        yield value;  // кусочек данных
    }
}

// Обрабатываем файл по частям
for await (const chunk of readHugeFile("big.bin")) {
    process(chunk);
}
```

## 13. Сравнение с PHP

| | PHP | JavaScript |
|---|---|---|
| Ключевое слово | `yield` | `yield` |
| Объявление функции | `function name()` | `function* name()` |
| Использование | `foreach ($gen as $val)` | `for...of` или `.next()` |
| Отправка значения внутрь | `yield` принимает через `$gen->send()` | `iterator.next(value)` |

В PHP генератор тоже возвращает объект `Generator`, у которого есть методы:
- `current()` — текущее значение
- `next()` — следующее
- `valid()` — есть ли ещё

В JS это скрыто за `for...of`, но можно использовать явно.

## 14. Проверочный код

```js
function* simpleGen() {
  yield 1;
  yield 2;
  yield 3;
}

const iter = simpleGen();

// У итератора нет свойства next как у связного списка
console.log(iter.next);  // [Function: next] — это метод!

// У него нет iter[0], iter[1]...
console.log(iter[0]);  // undefined — нет индексов!

// Только метод next(), который вычисляет значение
console.log(iter.next());  // { value: 1, done: false }
console.log(iter.next());  // { value: 2, done: false }
```
